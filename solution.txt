Суть уязвимости:
Приложение ожидает, что пользователь передаст имя файла (например, about.txt), и склеивает его с папкой public_docs.
Получается путь: public_docs/about.txt.
Однако в операционных системах можно с помощью ../ перейти на одну папку вверх.
Эксплуатация:
Файл flag.txt лежит в корне сервера (то есть на один уровень выше, чем папка public_docs).

Атакующий меняет значение параметра file в адресной строке на:
../flag.txt
Итоговый URL выглядит так:
http://127.0.0.1:5000/?file=../flag.txt
